## Exporting to CSV

CSV (Comma-Separated Values) is the universal format for data transfer.

### Basic CSV Export

**To export to CSV:**
1. Open report
2. Click **"Export"** â†’ **"CSV"**
3. CSV file downloads

**CSV characteristics:**
- Plain text format
- One file per table
- No formatting (colors, fonts)
- No charts or images
- Smallest file size

**[Screenshot: CSV export in text editor]**

### CSV Export Options

#### Delimiter Options

**Choose delimiter character:**
- **Comma** (,) - Standard, most compatible
- **Tab** (\t) - Better for data with commas
- **Semicolon** (;) - European Excel default
- **Pipe** (|) - Good for data with commas and quotes
- **Custom** - Specify any character

**When to use each:**
- Comma: Default for US/International
- Tab: Data contains commas (addresses, descriptions)
- Semicolon: European locales where comma is decimal separator
- Pipe: Maximum compatibility, rarely appears in data

#### Text Qualifier

**Enclose text values in:**
- **Double quotes** (") - Standard
- **Single quotes** (') - Alternative
- **None** - No text enclosure (risky with delimiters in data)

**Example with quotes:**
```csv
"First Name","Last Name","Company","Amount"
"John","Smith","Acme, Inc."," $50,000"
```

**Example without quotes:**
```csv
First Name,Last Name,Company,Amount
John,Smith,Acme Inc,50000
```

> **âš ï¸ Important**: Always use text qualifiers if your data might contain the delimiter character (e.g., commas in company names).

#### Encoding

**Choose character encoding:**
- **UTF-8** - Universal, supports all characters (recommended)
- **ASCII** - English only, maximum compatibility
- **Latin-1 (ISO-8859-1)** - Western European languages
- **UTF-16** - Required by some systems

**When to use:**
- UTF-8: Default, works for international characters
- ASCII: Importing to legacy systems
- Latin-1: European names and addresses
- UTF-16: Specific system requirements

#### Line Endings

**Choose line ending style:**
- **Windows** (CRLF: \r\n) - Windows systems
- **Unix/Mac** (LF: \n) - Mac, Linux, modern systems
- **Classic Mac** (CR: \r) - Old Mac OS (pre-OS X)

**Recommendation**: Use Windows (CRLF) for maximum compatibility unless importing to Unix/Mac-only system.

**[Screenshot: CSV export options dialog]**

### CSV Advanced Options

#### Header Row

**Header row options:**
- âœ“ Include header row (column names)
- Choose header style:
  - **Field names**: Technical names (first_name, last_name)
  - **Display labels**: User-friendly names (First Name, Last Name)
  - **Custom**: Specify custom headers

#### Null/Empty Values

**How to represent empty values:**
- **Empty string**: "" (nothing between delimiters)
- **NULL**: Literal text "NULL"
- **N/A**: Literal text "N/A"
- **Custom**: Specify custom placeholder

**Example:**
```csv
Name,Email,Phone
"John Smith","john@email.com",""          â† Empty string
"Jane Doe","",NULL                         â† NULL
"Bob Johnson","bob@email.com","N/A"       â† N/A
```

#### Number Formatting

**How to export numbers:**
- **Formatted**: As displayed (e.g., "$1,234.56")
- **Raw**: Unformatted value (e.g., "1234.56")

**Recommendation**: Use Raw for importing to other systems (no currency symbols or thousand separators).

#### Date Formatting

**Date format for CSV:**
- **ISO 8601**: YYYY-MM-DD (2026-01-17) - Recommended
- **US Format**: MM/DD/YYYY (01/17/2026)
- **European**: DD/MM/YYYY (17/01/2026)
- **Custom**: Specify format string

**Include time:**
- Date only: 2026-01-17
- Date and time: 2026-01-17 14:30:00
- ISO 8601 with timezone: 2026-01-17T14:30:00-05:00

**[Screenshot: CSV with various formatting options]**

### CSV Best Practices

> **ðŸ”„ Use for imports**: CSV is the best format for importing to other systems.
>
> **âœ… Test with small export**: Test delimiter and encoding with small sample before exporting large datasets.
>
> **ðŸ“… Use ISO date format**: YYYY-MM-DD is unambiguous across all locales.
>
> **ðŸ”¢ Export raw numbers**: Remove formatting for cleanest data import.
>
> **ðŸ’¾ UTF-8 encoding**: Handles international characters without issues.

**Consider your audience:**
- **Internal team**: Excel or CSV for manipulation
- **Executives**: PDF with executive template
- **External clients**: PDF with professional branding
- **Analysts**: Excel with formulas and pivot tables
- **Other systems**: CSV for maximum compatibility

**Consider the use case:**
- **Printing**: PDF with high quality, print-optimized layout
- **Presentations**: PNG charts at high resolution
- **Analysis**: Excel with formulas and formatting
- **Archiving**: PDF for long-term storage
- **Import**: CSV with clean, unformatted data

**Reduce export file size:**
- âœ“ Export only needed columns

- âœ“ Compress multiple exports into ZIP
- âœ“ Remove unnecessary charts from PDF

**File size targets:**
- Email attachments: < 10 MB (many systems limit at 10-25 MB)
- Mobile download: < 5 MB
- Quick share: < 2 MB
- Print: Size doesn't matter as much (prioritize quality)

### Data Security

- âœ“ Use password protection for confidential data
- âœ“ Add watermarks ("CONFIDENTIAL") to PDFs
- âœ“ Remove sensitive columns before exporting
- âœ“ Use secure file sharing instead of email for large/sensitive files
- âœ“ Set PDF permissions (disable printing/copying if needed)
- âœ“ Consider data masking for personally identifiable information (PII)
